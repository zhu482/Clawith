# 用户洞察系统

## 调度逻辑

根据研究目的，选择合适的洞察方法：

### 场景1: 数据分析与研究
**研究目的**：
- "用户研究"
- "用户画像"
- "行为分析"
- "需求分析"
- "痛点挖掘"
- "产品优化"

**核心需求**：通过数据和分析找到用户特征和优化方向

**调用**：`定量分析-数据驱动.md`

---

### 场景2: 真实反应模拟
**研究目的**：
- "帮我模拟用户反应"
- "用户会怎么看"
- "测试一下这个内容"
- "用户视角"
- "真实反馈"

**核心需求**：模拟真实用户的情绪和行为反应

**调用**：`定性共情-模拟用户.md`

---

### 场景3: 完整用户研究（同时调用）
**用户说**："做一个完整的用户研究"

**调用**：两者都调用
1. 定量分析：数据分析 + 用户画像
2. 定性共情：角色扮演 + 情绪模拟
3. 交叉验证，提供全面洞察

---

## 核心特点

### 共同点
- **用户中心**：一切以用户为中心
- **真实性**：基于真实数据和真实反应
- **可执行**：提供可落地的优化建议

### 区别点

| 维度 | 定量分析 | 定性共情 |
|------|---------|---------|
| **方法** | 数据驱动 | 角色扮演 |
| **核心工具** | 统计分析 + 用户画像 | 情绪模拟 + 行为预测 |
| **适用场景** | 需要数据支撑的决策 | 需要测试内容效果 |
| **输出** | 结构化研究报告 | 真实用户反应 |
| **视角** | 第三人称（客观分析） | 第一人称（用户视角） |

---

## 使用方式

**自动判断**：
```
用户："帮我分析一下用户画像"
系统：检测到数据分析需求 → 调用 定量分析-数据驱动.md
```

**指定方法**：
```
用户："用数据分析的方式研究用户"   → 调用 定量分析-数据驱动.md
用户："模拟用户看到这篇文章的反应" → 调用 定性共情-模拟用户.md
用户："全面研究用户"               → 同时调用两者
```

---

## 输入输出

### 定量分析
**输入**：
- 用户数据（行为数据、消费数据等）
- 研究问题（想了解什么）
- 业务背景（产品/行业信息）

**输出**：
- 用户画像
- 行为模式分析
- 痛点-爽点识别
- 优化建议（短中长期）
- 数据来源标注

### 定性共情
**输入**：
- 目标用户画像（年龄、职业、痛点）
- 待测试内容（文章、产品、策略）
- 使用场景

**输出**：
- 角色设定
- 第一反应（情绪）
- 消费过程（情绪变化）
- 最终决策（行为预测）
- 优化建议

## 协作技能

### 定量分析
- `search_web` - 查找行业报告和数据
- `retrieval` - 检索相关研究
- `concept-visualizer` - 生成用户画像图
- `memory` - 读取历史研究数据

### 定性共情
- `user-research-expert` - 补充用户画像
- `memory` - 读取目标受众信息
- `writing` - 优化内容

---

## 使用流程

### 定量分析流程
1. **接收数据**：用户提供数据或研究问题
2. **数据验证**：使用 search_web/retrieval 验证数据
3. **多维度分析**：
   - 行为偏好分析
   - 消费习惯研究
   - 心理特征洞察
   - 社交行为模式
4. **输出报告**：结构化研究报告
5. **标注来源**：所有数据来源标注

### 定性共情流程
1. **接收画像**：目标用户是谁
2. **进入角色**：完全"变成"那个用户
3. **模拟反应**：第一人称反馈
4. **行为预测**：预测用户行为
5. **给出建议**：优化建议

---

## 注意事项

### 定量分析红线
1. **数据必须真实**：使用 search_web 验证
2. **必须标注来源**：所有引用的数据
3. **结论必须有依据**：不能凭感觉
4. **建议必须可执行**：分短中长期

### 定性共情红线
1. **必须进入角色**：不能用AI口吻
2. **反馈必须真实**：不只说好话
3. **不能脱离角色**：避免专业术语
4. **行为预测要准**：基于心理学
