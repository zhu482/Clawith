# 记忆 - 个人记忆管理

识别、保存和使用用户的个人记忆，让AI写出来的东西**像用户说的话**，而不是像个机器人在背课文。

## 为什么需要记忆？

记忆是为了让AI写出来的东西**像你说的话**，而不是像个机器人在背课文。

风格可以学，但经历、观点、那些让你成为"贝塞尔砍刀"的细节，得靠记忆来喂。

## 使用方法

### 自动触发

AI会在对话中自动识别值得保存的记忆，并询问用户是否保存。

**强触发信号**（必须询问）：
- 用户讲了完整的故事（有时间、地点、过程、结果）
- 用户明确表达观点（"我觉得..."、"我认为..."、"我特别反感..."）
- 用户创造了新词或比喻（第一次出现的表达）
- 用户提到具体的项目经历、创业故事、踩坑过程

**弱触发信号**（可选询问）：
- 用户提到正在用的工具、技术
- 用户透露生活习惯、工作状态
- 用户的性格特点表现（自嘲、吐槽）

### 手动触发

用户可以主动要求保存记忆：
- "记下来"
- "保存到记忆"
- "记到记忆里"

## AI的记忆技能

### 1. 识别（在对话中自动检测）

**值得保存**：
- 具体的、有细节的经历（不是泛泛而谈）
- 明确的、可复用的观点（不是随口一说）
- 有画面感的表达方式（不是普通用词）
- 能体现人设的细节（不是流水账）

**不值得保存**：
- 日常闲聊（"今天天气不错"）
- 临时性的想法（"我可能会..."）
- 重复的内容（已经保存过的）
- 没有代表性的碎片信息

### 3. 询问（自然、轻松、不打扰）

**好的询问方式**：
- ✅ "这段经历挺有代表性的，要不要我帮你存到记忆里？以后写文章可以直接用。"
- ✅ "这个观点很有你的风格，要存下来吗？"
- ✅ "这个比喻不错，记下来？"

**不好的询问方式**：
- ❌ "检测到可保存信息，是否执行保存操作？"（太机械）
- ❌ "您的这段话非常有价值..."（太客套）

### 4. 保存（用户同意后）

**保存策略**：优先更新现有文件，没有对应类型才创建新文件

**第一步：检查现有记忆文件**
- 查看 `个人信息及记忆/记忆/` 目录
- 现有核心记忆文件：
  - `技能_能力边界.md` - 技能、工具栈、能力边界
  - `目标用户_心理画像.md` - 用户画像、痛点分析
  - `写作风格_核心特征.md` - 语言习惯、表达方式、风格特征

**第二步：判断保存方式**

1. **如果是技能/工具相关** → 更新 `技能_能力边界.md`
   - 正在用的工具、技术
   - 技能边界（会什么/不会什么）
   - 工作流程

2. **如果是用户理解相关** → 更新 `目标用户_心理画像.md`
   - 对用户痛点的新认知
   - 用户反馈和互动
   - 内容类型偏好

3. **如果是语言/风格相关** → 更新 `写作风格_核心特征.md`
   - 创造的新词、比喻
   - 反复使用的表达方式
   - 特定的吐槽句式

4. **如果是个人经历/观点** → 创建新文件
   - 文件路径：`个人信息及记忆/记忆/经历_[时间]_[事件].md`
   - 或：`个人信息及记忆/记忆/观点_[主题].md`

**第三步：执行保存**
1. 更新对应文件（或创建新文件）
2. 同时更新：`_记忆索引.md`
3. 如果是结构化信息，更新：`_实体数据.json`
4. 确认保存：简单告知用户"已保存到记忆"

### 4. 使用（写作时调用）

**写作时**：
- 读取记忆，找到相关的经历、观点、案例
- 让文章更有"你"的味道，而不是通用模板

**优化初稿时**：
- 对比初稿和记忆，看哪里可以加入你的个人经历
- 强化人设和风格

**选题深化时**：
- 从记忆中找到可以展开的角度
- 结合你的经历设计独特的叙事

## 记忆的分类与保存策略

### 现有核心记忆文件（优先更新）

**1. 技能_能力边界.md**
- 正在用的工具、技术
- 技能边界（会什么/不会什么）
- 工作流程
- 技术栈

**2. 目标用户_心理画像.md**
- 对用户痛点的认知
- 用户反馈和互动
- 内容类型偏好
- 用户行为观察

**3. 写作风格_核心特征.md**
- 创造的新词、比喻
- 反复使用的表达方式
- 特定的吐槽句式
- 语言习惯

### 需要创建新文件的类型

**1. 个人经历**
- 具体的项目、创业故事、踩坑过程
- 有时间、地点、过程、结果
- 有情绪波动的时刻
- 文件命名：`经历_[时间]_[事件].md`

**2. 观点和态度**
- 对某个现象/趋势的明确态度
- 反复强调的理念
- 对某类事物的吐槽或赞同
- 文件命名：`观点_[主题].md`

**3. 人设细节**
- 生活习惯、兴趣爱好
- 工作状态
- 性格特点
- 文件命名：`人设_[描述].md`

## 文件结构

```
个人信息及记忆/记忆/
├── _记忆索引.md                    # 记忆索引（必须更新）
├── _实体数据.json                  # 结构化数据（可选更新）
├── 技能_能力边界.md                # 核心文件（优先更新）
├── 目标用户_心理画像.md            # 核心文件（优先更新）
├── 写作风格_核心特征.md            # 核心文件（优先更新）
├── 经历_[时间]_[事件].md          # 按需创建
├── 观点_[主题].md                  # 按需创建
└── 人设_[描述].md                  # 按需创建
```

**核心原则**：
- 优先更新现有核心文件，保持记忆集中
- 只有个人经历、独立观点、人设细节才创建新文件
- 文件名要能一眼看出内容

---

## 核心原则

1. **真实性** - 记忆必须是真的，不是编的
2. **代表性** - 保存最能体现你风格的内容
3. **可用性** - 记忆要能直接用到写作中
4. **简洁性** - 不要记流水账，提炼核心

---

**记住**：记忆不是日记，是为了让AI更懂你。
