# 小红书排版 - 图文生成

将已经写好的小红书文稿转换为可直接发布的图文素材，包括封面图和内容图。

## 核心脚本与依赖关系

```
提示词模板 (prompt_templates.md)
    ↓
AI生图 (generate_ai_image.py) → 封面背景图
    ↓
封面生成 (cover_generator_v2.py) → 封面图 (背景+标题+头像)

内容图生成 (content_generator_html.js) → 内容图 (Markdown转图片)
```

### 脚本路径

所有脚本位于：`小红书/图文生成器/`

```
小红书/图文生成器/
├── prompt_templates.md         # 提示词模板（三种风格）
├── generate_ai_image.py        # AI生图脚本
├── cover_generator_v2.py       # 封面生成脚本（V2最新版）
├── content_generator_html.js   # 内容图生成脚本（HTML版，最新）
└── fonts/                      # 字体文件夹
```

### 脚本作用说明

1. **prompt_templates.md** - 提示词模板库
   - 提供三种封面风格：路透风、大字风、手绘风
   - 包含 Type A（完整封面）和 Type B（背景图素材）
   - **必须先选择合适的模板，才能生成高质量封面**

2. **generate_ai_image.py** - AI生图
   - 调用 Gemini 3 Pro Image 模型
   - 输入：Prompt（从模板中选择并填充）
   - 输出：封面背景图
   - **必须先完成这步，才能进行封面合成**

3. **cover_generator_v2.py** - 封面合成
   - 输入：背景图 + 标题 + 头像
   - 输出：完整封面（1440×2400）
   - 设计规范：背景图1200px，标题区1200px

4. **content_generator_html.js** - 内容图生成
   - 使用 Playwright + Chromium 渲染
   - 输入：Markdown 文稿
   - 输出：多张内容图（1440×2400）
   - 支持中文排版、Emoji、图片插入

## 前置检查

在开始生成图文之前，请确认：

1. **文稿文件存在** - 确保 Markdown 文稿已准备好
2. **Python 环境** - 确认 Python 3 已安装，且依赖库可用（PIL、requests）
3. **Node.js 环境** - 确认 Node.js 已安装（用于内容图生成）
4. **字体文件** - 确认 `小红书/图文生成器/fonts/` 下有字体文件
5. **头像文件** - 确认头像文件存在：`个人信息及记忆/素材/头像.jpg`
6. **输出目录** - 建议按文章标题建立独立文件夹（会自动清理特殊字符）

## 完整工作流

### 第一步：选择提示词模板

打开 `小红书/图文生成器/prompt_templates.md`，选择合适的风格。

#### 三种风格对比

| 风格 | 适用场景 | 视觉特点 |
|------|---------|---------|
| **风格1：路透风** | 职场、社会观察、深度纪实 | 纪实照片、低饱和度、严肃专业 |
| **风格2：大字风** | 观点输出、情绪共鸣、金句 | 纯文字排版、高对比、视觉冲击 |
| **风格3：手绘风** | 轻松故事、日常吐槽、幽默 | 火柴人插画、温暖、亲和力强 |

#### Type B（背景图素材）模板使用

**推荐使用 Type B**，因为可以精确控制文字排版。

**风格1（路透风）Type B 模板：**
```
A documentary-style photograph with a grainy film look. [从文章提取的核心场景描述]. The atmosphere is somber, cold, low-light, and heavily desaturated. No text overlay, no graphics, just the raw photograph.
```

**示例填充：**
```
A documentary-style photograph with a grainy film look. A lonely figure in a dark jacket viewed from behind, walking with head down past a closed automatic parking barrier gate at dusk in a typical Chinese residential complex. The atmosphere is somber, cold, low-light, and heavily desaturated. No text overlay, no graphics, just the raw photograph.
```

**风格3（手绘风）Type B 模板：**
```
A narrative minimalist line art sketch in a hand-drawn marker style on a beige background (#FAF3E1). An advanced stick figure wearing a blue t-shirt (#11659a) [动作], with an [具体表情，如：expressive sad/happy/tired] facial expression. The background ([场景]) is drawn with loose, simple black outlines. No borders, no comic frames, no text, full image composition, centered.
```

**示例填充：**
```
A narrative minimalist line art sketch in a hand-drawn marker style on a beige background (#FAF3E1). An advanced stick figure wearing a blue t-shirt (#11659a) holding a graphic card, with an expressive sad face and a sigh bubble. The background (a messy office desk) is drawn with loose, simple black outlines. No borders, no comic frames, no text, full image composition, centered.
```

> **注意**：风格2（大字风）只支持 Type A（AI直接生成），不适合模板合成。

### 第二步：生成封面背景图（AI生图）

使用填充好的 Prompt 调用 AI 生图脚本。

**Python 调用方式：**

```python
from 小红书.图文生成器.generate_ai_image import generate_image

# 使用第一步准备好的 Prompt
prompt = """A documentary-style photograph with a grainy film look.
A lonely figure in a dark jacket viewed from behind, walking with head down
past a closed automatic parking barrier gate at dusk in a typical Chinese
residential complex. The atmosphere is somber, cold, low-light, and heavily
desaturated. No text overlay, no graphics, just the raw photograph."""

# 生成背景图
bg_path = generate_image(
    prompt=prompt,
    output_path="小红书/图文生成器/output/ai-images/cover_bg.jpg"
)

print(f"✅ 背景图生成成功: {bg_path}")
```

**参数说明：**
- `prompt`：使用 Type B 模板填充后的完整 Prompt
- `output_path`：输出路径，建议放在 `output/ai-images/` 下
- `size`：可选，默认 1024x1024
- `style`：可选，模板中已包含风格描述，通常不需要额外指定

**关键点：**
- Prompt 必须来自 `prompt_templates.md` 的 Type B 模板
- 确保 Prompt 中包含 "No text overlay, no graphics"（不含文字）
- 生成的图片会被自动保存到指定路径

### 第三步：生成封面（封面合成）

使用背景图 + 标题 + 头像，合成完整封面。

**Python 调用方式：**

```python
from 小红书.图文生成器.cover_generator_v2 import CoverGeneratorV2

generator = CoverGeneratorV2()

# 从文稿标题提炼封面标题（8-15字，手动断行）
title = "211硕士转行大厂，\n被小区挡车杆\n打碎的\"精英梦\""

# 从文稿中提取精炼的副标题（脚本会自动折行）
subtitle = "大半年前，我也怀着一种\"老子不干了\"的孤勇，一头扎进创业的浪潮里"

# 生成封面
cover_path = generator.generate(
    bg_image_path=bg_path,  # 第二步生成的背景图
    title=title,
    subtitle=subtitle,
    avatar_path="个人信息及记忆/素材/头像.jpg",  # 固定路径
    output_path="小红书/输出目录/封面.jpg"
)

print(f"✅ 封面生成成功: {cover_path}")
```

**参数说明：**
- `bg_image_path`：第二步生成的背景图路径
- `title`：大标题，8-15字，用 `\n` 手动断行（2-3行效果最佳）
- `subtitle`：小标题，从文稿中提炼，脚本会自动折行
- `avatar_path`：头像路径，固定为 `个人信息及记忆/素材/头像.jpg`
- `output_path`：封面输出路径

**设计规范：**
- 画布：1440 × 2400
- 背景图：1440 × 1200（顶部，自动裁切居中）
- 大标题：字号120，梦源宋体-W22，最多3行
- 小标题：字号60，梦源宋体-W9，自动折行
- 作者信息：右下角，头像120×120

**标题提炼技巧：**
- 比文稿标题更短更抓眼
- 包含冲突/数字/情绪词
- 示例：
  - 文稿标题：《我为了显卡两个月工资，变成了完美牛马》
  - 封面标题：`为了显卡，\n我把自己练成了\n完美牛马`

### 第四步：生成内容图（HTML内容生成）

使用 Node.js 脚本将 Markdown 文稿转换为内容图。

**Bash 调用方式：**

```bash

  "小红书/文稿目录/文章标题.md" \

```

**参数说明：**
- 第一个参数：Markdown 文稿路径
- 第二个参数：输出目录（会生成 content_1.png, content_2.png...）

**依赖安装（首次使用）：**

```bash

npx playwright install chromium
```

**输出结果：**
- 生成多张内容图：`content_1.png`, `content_2.png`, `content_3.png`...
- 每张图片：1440 × 2400
- 自动分页，智能排版，保持信息密度合理

**Markdown 格式要求：**

```markdown
# 文章标题

## 第一个小标题

正文段落，支持普通文字。

**加粗的关键文字**会被突出显示。

(括号内的文字) 或（中文括号）会显示为灰色斜体。

## 第二个小标题

继续正文...

![图片描述](图片路径)

## 第三个小标题

更多内容...
```

**格式说明：**
- `#` 一级标题：文章总标题（通常不会显示在内容图中）
- `##` 二级标题：章节标题（会作为每章的大标题）
- 普通段落：正文内容
- `**文字**`：加粗强调
- `(文字)` 或 `（文字）`：灰色斜体注释
- `![](path)`：图片（会单独占一页）

**排版特性：**
- 每个 `##` 章节可以有多页
- 图片单独占一页，自动缩放居中
- 短页会上下居中显示
- 自动计算最佳分页，避免空白过多

4. 去除前后空格
5. 输出目录示例：`小红书/[选题名]/[清理后的标题]/`

### 依赖检查
- Python 脚本依赖：PIL、requests
- Node.js 脚本依赖：playwright（需要先安装 Chromium）
- 字体文件：`fonts/梦源宋体-W22.ttc`、`fonts/梦源宋体-W9.ttc`

### 提示词使用技巧
1. **必须使用模板**：不要自己写 Prompt，从 `prompt_templates.md` 选择
2. **Type B 优先**：除非是大字风，否则都用 Type B（背景图素材）
3. **场景具体化**：替换 `[动作]`、`[场景]` 时要具体描述，避免抽象
4. **保持风格一致**：同一系列文章建议使用同一风格
5. **统一角色形象**（手绘风）：所有插画中"我"的形象统一为
   ```
   A stick figure wearing a blue t-shirt (#11659a)
   ```
   （穿蓝色T恤的火柴人，颜色为 #11659a）

### 副标题提炼技巧
- **有具体场景**：描述一个可视化的场景，而非抽象概念
- **有情绪感受**：传递真实的情绪体验
- **有画面感**：让读者能在脑海中浮现画面
- **示例对比**：
  - ✅ 好的：`三个AI同时干活，我在旁边喝咖啡`（具体、有画面）
  - ✅ 好的：`当打工人的精神支柱，从理想变成了购物车里的那块显卡`（情绪、对比）
  - ❌ 不好的：`AI时代的新可能`（抽象、空泛）
  - ❌ 不好的：`关于职场的一些思考`（没有情绪、没有画面）

### 封面标题优化
- **长度控制**：8-15字最佳，最多20字
- **手动断行**：用 `\n` 控制断行，2-3行效果最好
- **视觉冲击**：包含数字、对比、冲突、情绪词
- **示例对比**：
  - ❌ 太长：`我在大厂工作一年后对职场内卷的深刻反思`
  - ✅ 合适：`大厂一年，\n我被内卷\n吃干抹净`

### 内容图排版建议
- **章节数量**：建议 3-5 个 `##` 章节
- **每章字数**：200-400 字，会自动分页
- **图片使用**：每篇最多 3 张插图，避免图片过多
- **总页数控制**：封面 + 5-9 张内容图为最佳

### 常见问题

**Q1: AI生图失败怎么办？**
- 检查 API Key 是否有效
- 检查网络连接
- 确认 Prompt 格式正确（从模板复制）

**Q2: 封面生成时背景图被裁切了？**
- 正常现象，脚本会自动居中裁切为 1440×1200
- 确保 AI 生成的图片是方形（1024×1024）

**Q3: 内容图生成报错 "排版拒绝：最后一页太空了"？**
- 说明某个章节内容太少，最后一页填充率低于 20%
- 建议增加 50-100 字，或者合并章节

**Q4: Node.js 脚本找不到字体？**
- 确认 `小红书/图文生成器/fonts/` 下有字体文件
- 确认当前工作目录在 `/Users/zhuzhiheng/Desktop/ZZ知识库`

**Q5: 文稿文件不存在怎么办？**
- 确认 Markdown 文稿已准备好
- 检查文件路径是否正确
- 如果是从长文拆分，确保先执行文稿拆分技能

**Q6: 图片路径错误怎么办？**
- 检查 Markdown 中的图片路径是否正确
- 确认图片文件确实存在
- 可以使用相对路径或绝对路径

**Q7: Python 脚本执行失败？**
- 检查 Python 环境和依赖库（PIL、requests）
- 查看具体错误信息
- 确认工作目录正确

**Q8: 输出目录创建失败？**
- 检查路径中是否包含特殊字符
- 使用路径清理规则去除非法字符
- 确认有写入权限
