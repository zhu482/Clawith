# 小红书内容生产 - 智能路由

这是小红书内容生产的总入口技能，根据用户需求自动路由到对应的子技能。

## 核心功能

这个技能负责识别用户需求，并调用对应的子技能完成任务：

```
用户需求
    ↓
智能识别 → 路由分发
    ↓
子技能1: 长文拆分成文稿
子技能2: 文稿生成图文素材
```

## 子流程 / 子技能说明

### 1. 长文拆分成文稿（长文 → 多篇小红书文稿）
**文件位置**：`.agent/skills/01-内容创作/小红书/图文/小红书文稿-长文拆分.md`

**功能**：将一篇长文（公众号文章、复盘等）拆分成多篇小红书文稿。

**适用场景**：
- 有一篇长文，想改造成一组小红书图文
- 想把系列内容拆成多篇小红书连载
- 需要先生成文稿，再做图文排版

**输入**：长文 Markdown 文件
**输出**：多篇小红书文稿（每篇约600字）

**触发关键词**：
- "拆分长文"
- "改成小红书文稿"
- "拆成多篇"
- "小红书连载"
- "长文改小红书"

### 2. 文稿生成图文素材（小红书文稿 → 封面图 + 内容图）
**文件位置**：`.agent/skills/01-内容创作/小红书/图文/小红书排版-图文生成.md`

**功能**：将小红书文稿转换为可直接发布的图文素材：每篇生成封面图（`cover.jpg`）+ 若干张内容图（`content_*.png`），并输出封面英文 Prompt（`cover_prompt.txt`，可选）。

**适用场景**：
- 已经有小红书文稿，需要生成图文
- 需要生成封面图
- 需要生成内容图

**输入**：小红书文稿 Markdown 文件
**输出**：封面图（1张）+ 内容图（若干张）

**触发关键词**：
- "生成图文"
- "生成封面"
- "生成内容图"
- "小红书排版"
- "做成图片"

## 路由规则

### 场景1：用户有长文，想改成小红书（长文拆分）
**判断依据**：
- 用户提到"长文"、"公众号文章"、"复盘"
- 用户说"拆分"、"改成小红书"
- 用户提供的是单篇较长的文章

**路由到**：长文拆分成文稿（《小红书文稿-长文拆分》流程）  
**执行说明**：由子文档《小红书文稿-长文拆分》负责具体的分析、拆分与生成文稿，本技能只负责在「长文 → 文稿 → 图文」之间衔接，尤其是拆分后询问是否进入场景2。

### 场景2：用户有文稿，想生成图文（图文生成技能）
**判断依据**：
- 用户提到"生成图文"、"生成封面"
- 用户已经有文稿文件
- 用户说"排版"、"做成图片"

**路由到**：文稿生成图文素材（`xiaohongshu-graphic` 技能）  
**执行说明**：将用户指定的文稿与图文生成脚本对接，由 `xiaohongshu-graphic` 子技能完成封面与内容图的全部细节，本技能只负责触发调用与结果汇总。

### 场景3：用户从零开始（需澄清素材形态）
**判断依据**：
- 用户说"我想做小红书"、"做一组小红书"
- 用户没有明确的长文或文稿
- 用户描述了选题或主题

**路由策略**：
1. 询问用户：是否已有长文？
   - **有长文** → 路由到场景1
   - **没有长文，有文稿** → 路由到场景2
   - **都没有** → 提示用户先准备内容或使用其他写作技能

### 场景4：一条龙服务（长文 → 文稿 → 图文）
**判断依据**：
- 用户明确说"从头到尾"、"一条龙"、"全流程"
- 用户希望从长文一直做到图文成品

**路由策略**：先调用《小红书文稿-长文拆分》产出多篇文稿并由用户确认，再批量调用 `xiaohongshu-graphic` 为每篇生成图文；本技能负责串联两个子流程、控制节奏和对用户做整体进度反馈。

```

## 路由决策树

```
用户请求
    ↓
是否提到"拆分"/"长文"?
    ├─ 是 → 场景1: 长文拆分
    │        └─ 完成后询问是否生成图文
    │             ├─ 是 → 场景2
    │             └─ 否 → 结束
    │
    └─ 否 → 是否提到"图文"/"封面"/"排版"?
         ├─ 是 → 场景2: 文稿生成图文
         │
         └─ 否 → 是否提到"一条龙"/"全流程"?
              ├─ 是 → 场景4: 一条龙服务
              │
              └─ 否 → 询问用户意图
                   ├─ 有长文 → 场景1
                   ├─ 有文稿 → 场景2
                   └─ 都没有 → 提示先准备内容
```

## 注意事项

### 路由优先级（当多个条件同时满足时）
1. **明确关键词** > 模糊意图
2. **长文拆分** > 图文生成（有长文时先拆分）
3. **用户明确指定** > 智能判断

### 技能调用规范
- 明确告知用户当前调用的子技能
- 在子技能执行前，简要说明将要做什么
- 子技能执行完成后，询问是否继续下一步

### 文件路径约定
- 长文通常在 `高质样例/` 或 `创作空间/` 下
- 小红书输出统一在 `小红书/[选题名]/` 下
- 头像固定路径：`个人信息及记忆/素材/头像.jpg`

### 典型目录结构（长文拆分 + 图文生成一体）
 
完整的目录与文件组织结构已经分别在《小红书文稿-长文拆分》和《小红书排版-图文生成》中详细说明，这里不重复列出代码块。  
本技能只需确保：长文拆分输出的文稿目录，能够被图文生成技能直接作为输入使用（路径约定保持一致）。
 
## 补充说明

- **先问清楚，再路由**：当用户只说「帮我做小红书」时，优先澄清现在手里是长文、现成文稿，还是只是一个选题想法，然后再套用上面的四个场景。
- **批量按篇推进**：处理多篇文稿时，按「一篇一完成」的节奏依次调用子技能，并在每篇完成后向用户报告进度（如：`第X篇 / 共Y篇`）。
