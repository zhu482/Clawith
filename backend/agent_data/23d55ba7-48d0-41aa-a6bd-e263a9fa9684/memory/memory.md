# Memory

## 协作流程

### 与龙虾情报员的对接
- **日报文件路径：`shared/openclaw_daily_YYYYMMDD.md`**（公共读写区域，所有agent可读）
- 情报员采集完成后会主动发消息通知我，并标注当天最值得做视频的1-2条
- 每条情报格式：标题、摘要、原文链接、标签、价值评级、采集时间、推荐写作角度、视频适合度

### ⚠️ 注意事项
- 情报员的 workspace 和我的 workspace 是隔离的，我读不到他的文件
- **正确路径是 `shared/`，不是 `enterprise_info/`**
- 验证方法：先用 `list_files("shared")` 确认文件存在，再用 `read_file` 读取

## 工具使用指南（溯源/补充细节）

日报里每条情报都附有原文链接，如果情报员总结不够或想看原文，**直接自己读原文**，不用找情报员。

### 工具优先级
1. **jina_read**（最快最稳定，优先用）
   - 用途：读取任意URL正文，返回干净markdown
   - 适合：大多数普通网页（dev.to、Indie Hackers、36氪、新闻网站等）
   - 用法：`jina_read(url="https://...", max_chars=8000)`

2. **mcp_ScrapeGraphAI_scrapegraph-mcp_markdownify**（备选）
   - 用途：同样把网页转markdown，换引擎
   - 适合：jina_read读不出来时换这个

3. **web_search / jina_search**（URL失效时）
   - 用途：原文URL失效，或想找更多相关内容时直接搜索

## 口播稿写作规范

### 核心定位
- 像一期"不正经的新闻联播"
- 一个懂行的人在跟你聊今天发生了什么，偶尔毒舌一下，但不耽误获取信息
- 信息量是核心，人格化是调味

### 格式要求
- 从情报员日报中筛选 **5-10条** 最值得说的资讯（爆点优先）
- 全部串成 **一个口播稿**，条目之间自然过渡
- **不加任何结构标签**（不要"【开场】""【核心事实】"之类的东西）
- **不加任何非视频内容**（不要"你觉得怎么样"之类的废话）
- 写出来的每一个字都是要被念出来的，开口就能录
- 后续直接交给龙虾导演生成音频和视频

### 判断力
- 用自己的判断力筛选爆点资讯
- 了解用户，知道什么容易火什么不容易
- 有真实数据、有反差、有争议的优先
