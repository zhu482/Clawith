## 一、商业创业

### B1 · NoDesk AI：两周极限转型，DeskClaw获近亿元融资
**来源**：36氪 · 2026-03-09 · https://36kr.com/p/3714634245009800
**记者**：邓咏仪 / 编辑：苏建勋

**人物背景**：NoDesk AI创始人宋健，从阿里、腾讯出来后，2014年创办电商DaaS公司"多准数据"（做电商消费者洞察和大数据精准营销），在SaaS寒冬前完成并购退出，后在智谱AI等公司任职探索基础模型到Agent应用落地。再度创业做NoDesk AI，主业是一站式品牌营销AI增长引擎（电商内容种草→投放→成交闭环）。

**DeskClaw怎么来的**：2025年春节前OpenClaw爆火，宋健带团队用两周开发了DeskClaw。起点是内部需求——让自己的电商Agent业务团队基于OpenClaw提升效率。开发完成后感受到外部需求，2026年2月14日个人版第一个版本上线，3月7日企业版发布并开源。

**产品形态**：一只在用户电脑上爬来爬去的"小螃蟹"AI助手。能操作浏览器、读写文件、调用本地应用，替用户完成市场调研、信息整理等工作。深度适配飞书、钉钉、企微，支持MiniMax、Kimi、GLM、Qwen、DeepSeek等多家模型，数据本地运行不上云。

**内部使用样本**：NoDesk飞书群挂满几十个机器人。AI PMO"小金桔"挂一堆Skill，@它即可自动总结群聊、创建飞书任务、更新项目文档，甚至把任务写进群公告。宋健说："我们PMO发自内心觉得在它面前确实是弟弟。"

**融资情况**：刚完成近亿元人民币融资，投资方为湖畔山南、初心资本、高途、顺福资本（四家共同投资，原文未披露轮次和估值）。

**团队状态**：00后占比超60%，春节全员不放假，人均每天1.5亿Token起步，相当于每人每天5000-10000行代码，"按秒开发，按分钟产品迭代，按天自我进化"。

**宋健金句**："已经不能用人类时间来衡量AI时代的进度。""这是最像iPhone的东西，第一次拿到像iPhone这样的产品，一下拉近了AI产品和大众的距离。""All in One这件事情之所以能成立，就是因为AI在这件事情上是无损的。"

## 三、安全预警

### D1 · 【高危已修复】ClawJacked：任意网站可完全劫持本地OpenClaw Agent
**来源**：Oasis Security · 2026-02-26发布 / 2026-03-09更新 · https://www.oasis.security/blog/openclaw-vulnerability
**修复版本**：v2026.2.25（24小时内修复）· **当前最新版v2026.3.7已包含修复**

**漏洞原理（完整攻击链5步）**：
1. 受害者用浏览器访问任意攻击者控制的网站
2. 网页JavaScript向localhost的OpenClaw网关端口发起WebSocket连接（**WebSocket连接localhost不受跨域策略限制**，浏览器不阻断）
3. 脚本暴力破解网关密码，速度数百次/秒——**网关的速率限制对localhost连接完全豁免**，失败次数不计数、不节流、不记日志
4. 密码破解成功后，脚本静默注册为受信任设备——**网关对来自localhost的新设备配对请求自动批准，无需用户确认**
5. 攻击者获得完全控制：可与AI Agent交互、导出配置数据、枚举已连接设备、读取日志

**危害**：OpenClaw配置文件中通常存有密码、API Key、消息内容等高度敏感信息。Agent有权限操作浏览器、读写文件、发送消息，攻击者获得控制权后等同于接管用户整台电脑的AI操作权限。

**背景补充**：Oasis还指出，大量OpenClaw安装属于"影子AI"——开发者自行安装，IT部门不知情，无集中治理，风险更高。

**建议**：立即升级至v2026.2.25或更高版本（当前最新v2026.3.7）。

### D2 · 【持续活跃】伪装OpenClaw安装包传播Infostealer + GhostSocks
**来源**：Security Boulevard / Huntress研究报告 · 2026-03-05 · https://securityboulevard.com/2026/03/latest-openclaw-security-risk-fake-github-repositories-used-to-deploy-infostealers/

**事件经过**：攻击活动运行于2026年2月2日至10日。Huntress研究员Jai Minton和Ryan Dowd于2月9日发现——一名用户下载了GitHub上伪装成"OpenClaw Installer for Windows"的安装包并运行。GitHub已下架该仓库和背后的组织。

**恶意软件组合**：
- **Steal Packer**（打包器）：负责分发Infostealer
- **GhostSocks**：代理恶意软件，让攻击者通过受害者的住宅网络路由流量，绕过IP封锁等防护
- **Windows系统**：部署GhostSocks
- **macOS系统**：部署AMOS（Atomic macOS Stealer）

**为何难以发现**：伪装安装包外观高度仿真，普通用户难以区分。OpenClaw本身配置文件含密码、API Key、消息内容，一旦被窃取危害极大。

**Huntress研究员原话**："每一种新的流行技术或全球性变化，都会吸引威胁行为者利用它来窃取凭证并出售访问权限——OpenClaw也不例外。"

**建议**：只从官方GitHub（github.com/openclaw/openclaw）下载，验证文件哈希值。

### E1 · Jensen Huang：OpenClaw是"可能有史以来最重要的软件发布"
**来源**：Thunder Tiger Europe（转述Morgan Stanley TMT大会发言）· 2026-03-09 · https://thundertiger-europe.com/nvidia-ceo-jensen-huang-openclaw-agentic-ai-release/

**发言场合**：Morgan Stanley TMT大会

**核心数据**：
- OpenClaw在**3周内**达到Linux操作系统花了**30年**才达到的采用量
- 单个Agentic任务消耗的Token是标准聊天提示的**1000倍**
- 持续运行的Agent（如7×24小时网络安全监控）最终Token消耗可能达到今日聊天交互的**100万倍**

**黄仁勋的"计算真空"理论**：标准聊天是短暂、低算力的；Agent在解决问题时会持续思考、规划、执行、检查、重试，这个迭代过程消耗资源量级完全不同。他将这种算力需求激增称为"compute vacuum"（计算真空），认为整个行业需要重新思考面向下一代智能系统的基础设施。

**英伟达自身已在用**：黄仁勋透露，英伟达内部已在使用OpenClaw Agent协助软件开发和工具创建，并观察到显著的生产力提升。

**采用曲线描述**：黄仁勋说OpenClaw的增长曲线"看起来几乎像Y轴本身"——接近垂直。
