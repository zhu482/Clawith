# 微信公众号排版

## 排版风格

**极客红黑风 (Geek Chic Style)**

### 配色方案
- **主色**：`#202124` (黑)
- **强调色**：`#ab1942` (深红)
- **背景色**：`#FFFFFF` (白)
- **灰色**：`#666666` (注释文字)
- **引用背景**：`#F4F5F7` (浅灰)

### 标题样式
- **一级标题**：居中 + 底部深红下划线
- **二级标题**：左侧深红竖线 + 无缩进
- **三级标题**：带方形红色 Icon

### 特殊处理
- 加粗文字：`font-weight: 600`（避免太黑）
- 引用块：灰色圆角方块，无左侧红线
- 注释：`(文案)` 或 `（文案）` 转为灰色斜体

## 排版方法论

排版不只是样式，更是**阅读体验设计**（人机交互）。

### 核心原则

**把文章当产品，把排版当交互设计**
- 用户从标题 → 封面 → 第一屏 → 全文的旅程，每一步都要设计
- 目标：让跳读用户也能看懂全文

### 4个关键技巧

#### 1. 信息密度要错落有致
- 像电影节奏，不能一直高密度轰炸
- 密集段落（观点、论证）和留白段落（过渡、呼吸）交替
- **短句优先**：每句4-8字为佳，标点符号前后超过十几字就难读

#### 2. 每3-5行标一个重点句
- **目的不是标"核心观点"，而是让跳读用户也能看懂**
- 用户只读加粗句，也能理解文章主旨
- 这是当前用户阅读习惯决定的（完读率思维）

#### 3. 说人话验证法
- **标准**：能不能通顺念出来？
- 念的时候嘴打架 = 不是人话
- 短视频脚本和公众号差异：口语化程度不同，但都要"能念"

#### 4. 专业词汇的度
- 完全没有 = 太水，让人没有收获感
- 太多 = 读者放弃，产生挫败感
- **技巧**：核心观点处保留专业词汇，过渡处用口语化表达

### 完读率参考

| 完读率 | 评价 |
|-------|------|
| < 40% | 较差，需要优化排版和节奏 |
| 40-50% | 一般 |
| > 50% | 较好 |
| > 60% | 优秀 |

### 反模式（避免）

- ❌ 一大段文字不分段（视觉摩擦阻力大）
- ❌ 花哨排版（颜色、动效、跳动元素）
- ❌ 连续讲道理超过3段没有锚点
- ❌ 长句超过15个字不断句

## 技术细节

### 图片处理
- **自动读取**：Markdown 中引用的本地图片
- **Base64 编码**：转为 Base64 内嵌到 HTML 中
- **解决裂图**：彻底解决跨平台粘贴"裂图"问题

### CSS 内联
- 所有样式内联到 HTML 标签中
- 无需外部 CSS 文件
- 粘贴即可保持格式

### 智能清洗
- **去油腻加粗**：加粗内容超过 15 字自动取消加粗
- **段落合并**：自动识别 Markdown 换行，合并碎片文本
- **注释变灰**：`(文案)` 格式自动转为灰色斜体

## 注意事项

1. **图片大小**：微信公众号对图片大小有限制，建议压缩后再使用
2. **字数限制**：微信公众号有字数限制（约 20000 字）
3. **测试预览**：粘贴后在公众号后台预览，确认格式无误
4. **链接处理**：Markdown 中的链接会保留，但公众号需要白名单才能跳转

## 常见问题

### Q: 粘贴后格式丢失？
A: 确保使用"粘贴"而不是"粘贴为纯文本"。Mac 上是 Cmd+V，Windows 上是 Ctrl+V。

### Q: 图片显示不出来？
A: 检查 Markdown 中的图片路径是否正确，脚本会自动将本地图片转为 Base64。

### Q: 加粗太粗/太黑？
A: 脚本已优化为 `font-weight: 600`，如果还是太粗，检查是否有多层加粗。

### Q: 引用块左侧有红线？
A: v2.0 已去掉左侧红线，如果还有，检查脚本版本。
