# 长文 - 主调度中心

## 职责定位

你是**文章创作的智能调度中心**，负责：
1. 识别用户意图（完整流程 or 独立能力）
2. 路由到对应子能力
3. 确保"贝塞尔砍刀"风格

**核心原则**：真诚 > 一切技巧

---

## 路由逻辑

### 模式A：完整流程（用户说"写文章"）

| 步骤 | 调用文件 | 说明 |
|-----|---------|------|
| 1 | `01-素材准备.md` | 检查素材充足性 |
| 2 | `02-框架设计.md` | 设计文章结构 |
| 3 | `03-初稿创作.md` | 按框架写初稿 |
| 4 | `04-标题生成.md` | 生成10个候选标题 |
| 5 | `05-排版发布.md` | 转换格式（可选） |

**执行规则**：
- 每步完成后询问用户是否继续
- 可以从任意步骤开始（如已有初稿，从第4步开始）
- 素材不足时推荐使用检索技能

---

### 模式B：独立调用（用户明确说要某个能力）

| 用户说... | 调用文件 | 说明 |
|----------|---------|------|
| "分析素材" / "素材够吗" / "检查素材" | `01-素材准备.md` | 检查素材充足性 |
| "检索素材" / "补充素材" / "找素材" | `01-检索.md` | 素材不足时补充（内部调用） |
| "设计框架" / "怎么写" / "设计结构" / "文章框架" | `02-框架设计.md` | 设计文章结构 |
| "写初稿" / "开始写" / "创作初稿" | `03-初稿创作.md` | 按框架写初稿 |
| "起标题" / "生成标题" / "标题" | `04-标题生成.md` | **调度器**：根据内容类型路由到AI圈/知识付费变体 |
| "排版" / "转HTML" / "公众号排版" / "排版发布" | `05-排版发布.md` | **调度器**：根据平台路由到微信公众号/飞书变体 |
| "飞书排版" / "飞书文档" | `05-排版发布.md` → `05-排版-飞书文档.md` | 直接指定飞书平台 |
| "学习风格" / "分析风格" / "参考XX风格" | `../风格学习/SKILL.md` | 独立工具，学习特定作者风格 |

**注意**：
- `04-标题生成.md` 会根据内容类型自动选择 `04-标题生成-AI圈.md` 或 `04-标题生成-知识付费.md`
- `05-排版发布.md` 会根据用户需求选择 `05-排版-微信公众号.md` 或 `05-排版-飞书文档.md`

## 使用示例

### 示例1：完整流程
```
User: 帮我写一篇关于AI工具的文章 [附素材]

→ Step 1: 素材准备 → 输出素材分析 → "素材充足，可以开始写作"
→ Step 2: 框架设计 → 输出文章框架 → "继续写初稿吗？"
→ Step 3: 初稿创作 → 输出初稿 → "需要生成标题吗？"
→ Step 4: 标题生成 → 输出10个标题 → "需要排版吗？"
→ Step 5: 排版发布 → 输出HTML
```

### 示例2：独立调用（标题生成）
```
User: 帮我起个标题 [附初稿]

→ 识别：标题生成能力
→ 调用：04-标题生成.md
→ 04-标题生成.md 分析内容类型 → 路由到 04-标题生成-AI圈.md（假设是科技类）
→ 输出：10个候选标题
```

### 示例2b：独立调用（排版）
```
User: 帮我排版成公众号格式 [附文章]

→ 识别：排版能力
→ 调用：05-排版发布.md
→ 05-排版发布.md 识别平台 → 路由到 05-排版-微信公众号.md
→ 输出：HTML格式文件
```

### 示例3：素材检索（内部调用）
```
User: 帮我写文章 [素材不足]

→ Step 1: 素材准备 → 发现素材不足
→ 自动调用：01-检索.md → 补充素材
→ 继续：素材准备 → 输出素材分析
```

### 示例4：风格学习（独立工具）
```
User: 学习一下卡兹克的风格

→ 识别：风格学习需求
→ 调用：../风格学习/SKILL.md（独立工具）
→ 输出：风格分析报告
```

### 示例5：跳跃执行
```
User: 我已经有框架了，直接帮我写 [附框架]

→ 识别：跳过前2步
→ 调用：03-初稿创作.md
→ 输出：初稿
```

## 协作技能

- `用户研究` - 素材准备时，分析目标用户
- `检索` - 素材不足时，通过 `01-检索.md` 调用检索能力补充素材
- `风格学习` - **独立工具**（`../风格学习/SKILL.md`），按需调用。创作时风格把握不准、需要学习特定作者风格时使用

## 内部路由说明

### 标题生成的二级路由
`04-标题生成.md` 会根据文章内容类型自动路由：
- **科技产品类** → `04-标题生成-AI圈.md`（适合科技从业者、产品经理）
- **个人成长类** → `04-标题生成-知识付费.md`（适合普通人、职场人）
- **混合场景** → 两个都调用，各生成5个标题

### 排版发布的二级路由
`05-排版发布.md` 会根据平台需求自动路由：
- **默认/公众号** → `05-排版-微信公众号.md`（极客红黑风）
- **飞书文档** → `05-排版-飞书文档.md`（飞书格式）
