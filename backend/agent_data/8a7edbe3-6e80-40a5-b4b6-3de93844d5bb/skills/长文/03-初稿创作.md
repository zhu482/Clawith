# 03 - 初稿创作

## 职责定位

按照框架写初稿，应用"贝塞尔砍刀"风格。

**核心问题**：写什么？怎么写？

**关键原则**：严格按照框架，不要偏离结构

---

## 执行流程

### 第一步：读取框架和写作方法

**读取文件**：
- `02_文章框架.md` - 文章结构
- 确认使用的写作方法（框架中会标注）

**确认**：
- 文章结构清晰
- 情绪推进明确
- 转折点已标注
- 使用的写作方法明确（默认：爆款文章方法论）

---

### 第二步：读取写作风格

**核心风格文件**（始终参考）：
- `个人信息及记忆/记忆/写作风格_核心特征.md`

**核心气质**：
- 真诚
- 黑色幽默
- 唤起深层情感
- 拒绝宏大叙事

**关键原则**：
- **如果一句话让你觉得"我不会这么说"，那就别写**

---

### 第三步：按框架创作

**创作要点**：

1. **严格按照结构**
   - 每个部分对应框架中的设计
   - 不要偏离核心观点

2. **开头写法**（最重要的3行）

   **核心原则**：
   - 开头要狠，直接切入，不寒暄不铺垫
   - 第一人称「我」开头，让读者进入故事
   - 3行内抛出钩子（反认知观点/痛点/悬念）

   **6种开头模板**（选择最适合的一种）：

   | 模板 | 结构 | 适用场景 |
   |-----|------|----------|
   | 发现切入 | "最近/前两天，我发现了一个..." + 直接抛结论 | 工具、技巧、干货类 |
   | 故事切入 | 时间+地点+人物+动作，电影镜头式 | 案例、经验分享类 |
   | 困惑切入 | "我经常看到很多朋友..." + 痛点描述 | 教程、心得类 |
   | 节点切入 | "今天，是XX的X周年/我刚刚..." | 复盘、总结类 |
   | 场景特写 | 具体场景描写，5秒内进入画面 | 人物故事、深度报道类 |
   | 对话切入 | "朋友给我打电话说..." | 真实案例、帮朋友解决问题类 |

   **示例**（仅供参考，不要照搬）：
   ```
   # 发现切入
   前两天，我在网上发现了一个很有趣的Prompt技巧。
   就是，通过重复输入提示词，可以将准确率从21%提高到97%。
   
   # 故事切入
   周日下午，正在家里看电影，突然接到老家一好哥们的电话。
   他说年底了，计划再开家新店。
   
   # 困惑切入
   已经2026年了，其实还是看到很多朋友，说不知道怎么能更好的跟AI对话。
   就是那种问点日常还可以，一旦稍微专业一点，效果就不行了。
   ```

3. **正文写法**
   - **情绪密度 > 知识密度**：用反问、自问自答制造对话感
   - **每3段插入锚点**：例子/数据/比喻/转折，避免连续讲道理
   - **扫读优化**：关键词加粗，用列表代替长段落

4. **标注转折点**
   - 用框架中设计的转折词
   - 让情绪推进自然

5. **应用风格**
   - 压缩语言（每句话都要精炼）
   - 制造金句（关键观点要有力）
   - 细节精准（选择最有画面感的细节）
   - 真诚表达（不装逼、不说教）

6. **文章内不写标题**
   - 标题在第5步生成
   - 初稿只写正文

---

### 第四步：风格自检（初步）

**第一优先级：真诚度**
- 这句话是我真实的想法吗？
- 我在装逼吗？
- 我在表演"有感悟"吗？

**第二优先级：共情度**
- 这段话能让读者想起自己的经历吗？
- 我说出了读者想说但说不出的话吗？

**第三优先级：风格**
- 这段话是在说教吗？
- 这个词是营销号用的吗？
- 我在云端还是在尘埃里？

---

### 第五步：生成初稿文件

**文件名**：`03_初稿.md`

**保存位置**：对应的选题文件夹下

**内容**：
- 完整正文（无标题）
- 按照框架结构
- 已应用风格

---

## 输出

- `03_初稿.md`
- 完整的文章初稿
