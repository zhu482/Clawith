# 小红书排版 - 图文生成

这个技能负责把**已经写好的小红书文稿**，变成可以直接发布的小红书图文素材：

- 每篇文稿 → 一个文件夹  
- 文件夹内有：封面图 `cover.jpg` + 若干张正文图 `content_01.png`、`content_02.png` …  
- 强调视觉风格统一、信息密度适中、适配手机竖屏阅读。

> 上游默认由《小红书文稿-长文拆分》提供文稿；也可以直接接收用户手写的单篇文稿。

---

## 一、输入与输出约定

### 1. 输入要求

- 单篇或多篇小红书文稿，推荐为 Markdown 格式：  
  - 一级标题：文章标题（`# 标题`）  
  - 正文：按小红书习惯分段（20-50 字一段）  
  - 单行加粗段落：`**情绪锤子/关键句**`，用于视觉强调  
- 可选元信息（如果有就用，没有就自己推断）：  
  - 选题名 / 系列名  
  - 期数（如「01」「02」）  
  - 目标人群 / 账号定位

### 2. 输出结构

以单篇文稿为例：

```bash
小红书/[选题名]/
└── [标题]/
    ├── cover_prompt.txt        # 封面图的英文Prompt（可选）
    ├── cover.jpg               # 封面图（竖版 4:5 或 3:4）
    ├── content_01.png          # 正文第一页
    ├── content_02.png
    └── content_03.png
```

多篇文稿时，每篇各一个子目录，命名规则与《小红书文稿-长文拆分》保持一致。

---

## 二、整体流程

完整流程分为 3 个阶段：

1. 解析文稿结构 → 切分为「视觉卡片」级别内容块  
2. 设计版式 → 确定每一张图上放哪些文字、使用什么布局  
3. 生成图片 → 调用图片模型 / 视觉引擎生成封面和内容图

### 流程示意

```text
小红书文稿（单篇）
    ↓
内容切片（按信息块拆分）
    ↓
图文版式设计（每张图的内容+布局）
    ↓
封面图 Prompt + 正文图 Prompt
    ↓
调用图像模型（如 Gemini 3 Pro Image）
    ↓
导出图片文件到对应目录
```

## 三、内容切片：把文稿拆成「图文卡片」

### 1. 切片原则

**一张图 = 一个清晰的信息锤子**，不要塞太多：

- 单张图内推荐文字总字数：  
  - 标题 + 正文合计 **不超过 80 字**，最佳 40-60 字  
- 同一张图只讲一件事 / 一个角度 / 一个步骤  
- 图与图之间要形成**节奏**：  
  - 情绪起伏（痛点 → 共鸣 → 方案 → 收获）  
  - 信息递进（Why → What → How → Case）

### 2. 切片操作步骤

1. 按段落遍历文稿  
2. 识别以下几类重要段落，优先单独成图：  
   - 单行加粗金句（`**...**`）  
   - 「三点」「四步」这类列表核心总结  
   - 痛点/自嘲/自我暴露类段落  
   - 关键案例或反转情节  
3. 对于解释性较强的段落，可以两段合并到一张图，只要总字数不超  
4. 给每个内容块打标签：`[HOOK]`、`[PAIN]`、`[INSIGHT]`、`[STORY]`、`[METHOD]`、`[SUMMARY]`

### 3. 切片结果示例

```text
Block 1 [HOOK]     → 图1：钩子+标题
Block 2 [PAIN]     → 图2：痛点放大
Block 3 [STORY]    → 图3：故事场景
Block 4 [METHOD]   → 图4：三步方法
Block 5 [SUMMARY]  → 图5：总结+行动号召
```

## 四、版式设计：每一张图长什么样

### 1. 画幅与风格

- **竖版优先**：  
  - 建议 4:5（1080×1350）或 3:4（1080×1440）  
  - 保证在小红书主流设备上显示舒服  
- 背景：以浅色 / 纯色为主，避免复杂纹理干扰文字阅读  
- 字体风格：  
  - 标题：粗体、大字号  
  - 正文：更细、更小一号  
  - 适度使用高亮色块/描边强调关键词

### 2. 信息层级

每一张图，至少要有两个层级：

1. **主信息**：一眼看到的标题或金句（8-18 字）  
2. **辅信息**：解释/步骤/案例（30-60 字）

可以按以下模板组合：

- 模板 A：大标题 + 小解释  
- 模板 B：标题 + 3 点列表  
- 模板 C：标题 + 左故事右要点  
- 模板 D：纯金句大字（中间 1 句 + 小署名）

### 3. 色彩与图形元素

- 全篇色彩尽量**控制在 2-3 种主色 + 1-2 种点缀色**  
- 常用组合：  
  - 黑 + 白 + 一种主题色（如红 / 蓝 / 橙）  
  - 暗底 + 亮字 或 亮底 + 深字（二选一，不要来回乱跳）  
- 按内容气质选择风格：  
  - 干货教程 → 干净、理性、结构化  
  - 情绪故事 → 有点手写感 / 贴纸感 / 胶带元素

## 五、封面图设计

封面图决定「点进率」，优先级极高。

### 1. 封面文本

从文稿中提炼一句**更适合封面的标题**，一般与内部标题略有不同：

- 内部标题可以稍长一点，解释更清楚  
- 封面标题要更短、更抓眼：  
  - 8-12 字最佳，上限不超过 15 字  
  - 尽量包含冲突/数字/情绪词

示例改写：

- 文稿标题：《我为了显卡两个月工资，变成了完美牛马》  
- 封面标题：`为了显卡，我把自己练成了完美牛马`

### 2. 封面画面内容

根据文稿气质，给出封面画面建议（文字说明即可，方便下游图像模型使用）：

- 主体人物 / 场景  
- 表情 / 动作  
- 前景/背景元素（如电脑、工位、深夜灯光）  
- 主色调（冷/暖/高对比）

### 3. 封面 Prompt（如需调用图像模型）

输出一个英文 Prompt（写入 `cover_prompt.txt`），但强制说明：**图片中的文字区域留空，由后续工具/人工加字**。

示例结构：

```text
Prompt for Gemini 3 Pro Image:
A cinematic illustration of an overworked Chinese office worker staring at a gaming graphics card on an e-commerce website at midnight.
Dark office background, only the screen and desk lamp are lighting the scene.
Mood: exhausted but obsessed, slightly humorous.
Aspect ratio 4:5, clean composition, leave empty space at the top for Chinese title text.
```

## 六、正文图生成逻辑

### 1. 单张图的文本结构

对每个内容块，生成结构化说明，方便排版：

```json
{
  "page_index": 1,
  "layout": "title_plus_body",
  "title": "完美牛马的自我驯化",
  "body": [
    "你有没有过这种体验：",
    "明明领导没逼你，你却会主动加班，把自己安排得明明白白。",
    "唯一的动力，是购物车里那块两个月工资的显卡。"
  ],
  "highlight": ["主动加班", "两个月工资的显卡"],
  "style_hint": "slightly ironic, dark background with warm light"
}
```

### 2. 页数控制

- 通常一篇小红书文稿控制在 **5-9 张图**：  
  - 1 张封面 + 4-8 张正文图  
- 如果内容特别多：  
  - 优先保证逻辑完整；  
  - 适当合并信息块，压缩为信息密度稍高的页面；  
  - 超过 12 张图时，要提醒用户考虑拆成上下篇。

### 3. 与风格/视觉技能协同

- 如果用户明确提到「卡片风」「白板手绘」「知识卡片」等：  
  - 可以建议调用《概念可视化》相关技能生成更专业的配图  
  - 当前技能以**小红书图文排版**为主，不做复杂逻辑图

## 七、与《小红书文稿-长文拆分》的协同方式

从长文到图文小红书，推荐采用双技能协同：

```text
长文 → 小红书文稿-长文拆分 → 用户确认拆分方案
    → 生成多篇小红书文稿
    → 小红书排版-图文生成 → 每篇产出封面 + 内容图
```

协同约定：

- 文稿拆分阶段负责：  
  - 控制每篇字数在 500-800 字  
  - 设计合适的小标题与结构  
  - 保证故事/观点的完整性
- 图文生成阶段负责：  
  - 控制单页信息量  
  - 设计视觉节奏与图文结构  
  - 输出封面 + 内容图 +（可选）图像模型 Prompt

---

## 八、质量检查清单

在认为「可以交付」之前，至少自检：

- [ ] 每张图的文字在手机上阅读不会显得太挤（不超过 80 字）  
- [ ] 同一篇内的字体风格、颜色体系大致统一  
- [ ] 封面标题足够抓眼，且与内容匹配  
- [ ] 情绪/故事型内容有画面感，不是纯干巴巴的论述  
- [ ] 教程/方法型内容有步骤或要点，方便截图收藏  
- [ ] 图数在 5-9 张之间，如超过 12 张已提醒用户考虑拆分  
- [ ] 所有输出文件路径命名干净（无特殊字符，长度适中）
