# IP打造系统

## 调度逻辑

根据用户需求，选择合适的打造方法：

### 场景1: IP人设设计
**需求特征**：
- "个人IP打造"
- "人设设计"
- "个人品牌"
- "找到我的定位"
- "我适合做什么人设"

**核心需求**：找到独特人设，建立个人品牌

**调用**：`IP架构师-人设设计.md`

---

### 场景2: 知识变现路径
**需求特征**：
- "知识变现"
- "如何赚钱"
- "变现路径"
- "课程怎么卖"
- "粉丝转化"

**核心需求**：找到最短变现路径，把知识变成收入

**调用**：`知识变现-商业路径.md`

---

### 场景3: 完整IP打造（同时调用）
**用户说**："帮我从0到1打造个人IP"

**调用**：两者都调用
1. IP架构师：人设设计 + 差异化定位
2. 知识变现：变现路径 + 商业模式
3. 输出完整的IP打造方案

---

## 核心特点

### 共同点
- **真实性**：基于真实特质，不是虚假包装
- **商业化**：一切以变现为导向
- **可执行**：提供具体的行动路径
- **系统化**：从定位到变现的完整链路

### 区别点

| 维度 | IP架构师 | 知识变现导师 |
|------|---------|-------------|
| **切入点** | 人设设计 | 商业路径 |
| **核心工具** | DNA人设定位法 | 三板斧（诊断-设计-优化） |
| **适用场景** | 不知道做什么人设 | 不知道怎么赚钱 |
| **输出** | 人设方案 + 内容策略 | 变现路径 + 商业模式 |
| **时间维度** | 长期（IP建设） | 短期（快速见效） |

## 使用方式

**自动判断**：
```
用户："我想做个人IP，但不知道做什么人设"
系统：检测到人设设计需求 → 调用 IP架构师-人设设计.md
```

**指定方法**：
```
用户："帮我设计一个人设"         → 调用 IP架构师-人设设计.md

```

## 输入输出

### IP架构师
**输入**：
- 个人背景（经历、技能、性格）
- 目标受众
- 现有资源

**输出**：
- 人格基因诊断
- 差异化标签
- 人设故事包装
- 内容策略
- 传播符号设计

### 知识变现导师
**输入**：
- 专业领域 / 技能特长
- 现有粉丝 / 影响力
- 变现目标

**输出**：
- 知识富矿识别
- 产品设计（引流版-转化版-变现版）
- 定价策略
- 转化路径
- 快速验证方案

## 协作技能

### IP架构师
- `memory` - 读取个人背景和经历
- `user-research-expert` - 分析目标受众
- `writing` - 优化内容策略
- `concept-visualizer` - 可视化人设定位

### 知识变现导师
- `ip-architect` - 帮助设计人设
- `business-model-surgeon` - 优化商业模式
- `title-system` - 生成课程标题
- `search_web` - 查找同赛道案例

## 使用流程

### IP架构师流程
1. **透视人格基因**：挖掘真实特质
2. **设计差异化标签**：DNA人设定位法
   - D: 差异化标签
   - N: 故事包装
   - A: 放大传播
3. **构建内容策略**：持续强化人设
4. **输出完整方案**：从定位到执行

### 知识变现导师流程
1. **认知诊断**：找到知识富矿
   - 你最擅长什么？
   - 用户最需要什么？
   - 两者的交集在哪？
2. **产品设计**：系统化包装
   - 引流版：免费内容
   - 转化版：低价产品
   - 变现版：高价服务
3. **执行优化**：快速验证
   - 最小可行产品（MVP）
   - 快速测试反馈
   - 迭代优化

## 注意事项

### IP打造红线
1. **必须真实**：不是虚假包装，是真实特质放大
2. **必须差异化**：不是模仿别人，是找到独特性
3. **必须可持续**：不是短期热度，是长期IP
4. **必须有商业价值**：不是自嗨，是能变现

### 知识变现红线
1. **价值先行**：先提供价值，再谈变现
2. **用户思维**：站在用户角度包装知识
3. **快速验证**：用MVP测试市场
4. **系统思维**：不是单品，是产品矩阵
