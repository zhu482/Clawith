## 【一、版本更新：备份工具上线了，终于】

OpenClaw 2026.3.8 发布了。GitHub Stars 295K，上期是284K，一周涨了11K。

这次更新我最关注的是两件事：

第一，**备份工具**。终于有了。`openclaw backup create`，可以在你做破坏性操作之前强制提醒你备份。听起来很基础，但我知道有多少人在没有备份的情况下把自己的工作流搞崩过。

第二，**ACP身份验证**。多Agent工作流里，Agent现在可以验证"跟我说话的那个Agent是不是真的是它"。这个功能听起来很技术，但背后的问题很简单：如果你的工作流里有十几个Agent互相通信，你怎么保证没有人在中间假冒？现在有答案了。

另外还修了Telegram重复触发的bug，加了Talk模式静默超时配置。都是小事，但说明在认真维护。

## 【二、安全预警：先别往下看，去检查一下你的实例】

好，现在说那件你可能已经中招的事。

**CVE-2026-25253。**

Hunt.io 扫描发现，互联网上有超过 **17,500个** OpenClaw实例暴露在公网。漏洞是什么？`/api/export-auth` 这个接口，没有任何认证，没有任何授权检查，任何人都可以直接调用它，把你存储的所有AI服务API密钥——OpenAI的、Anthropic的——全部导出走。

17,500个实例。

我不知道你的实例是不是其中之一，但如果你把OpenClaw Gateway直接暴露在公网上，现在去检查一下。不是明天，是现在。

还有一件事：有人在Bing AI搜索里投放了假的OpenClaw安装包。你搜"OpenClaw Windows"，Bing AI会推荐一个看起来很正常的GitHub链接，点进去下载，装的是GhostSocks恶意软件。这个攻击从2月9日就开始了，现在还在跑。

——所以，如果你最近帮朋友装OpenClaw，或者你自己在Windows上装过，确认一下来源是官方的。
